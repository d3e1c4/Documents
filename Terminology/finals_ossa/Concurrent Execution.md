අපි හිතමු ඔයා එකම වෙලාවේ **Chrome** එකේ වීඩියෝ එකක් බලන ගමන්, **Word** document එකක් ටයිප් කරනවා කියලා. ඔයාට හිතෙන්නේ මේ වැඩ දෙකම එකම වෙලාවේ (Simultaneously) සිද්ධ වෙනවා කියලා. හැබැයි ඇත්තටම පරිගණකය ඇතුළේ වෙන්නේ වෙනස් දෙයක්.

මේක පැහැදිලි කරගන්න මේ කරුණු බලන්න:

### 1. Concurrency (සමාන්තරව වගේ පෙනීම)

පරිගණකයක තියෙන **CPU** එකට එක වතාවකදී කරන්න පුළුවන් එක Instruction එකක් විතරයි. හැබැයි CPU එක හරිම වේගවත්. ඒ නිසා එයා කරන්නේ මේ වගේ දෙයක්:

- තත්පරයෙන් ඉතාම කුඩා කොටසක් (Milliseconds) Chrome එකට දෙනවා.
    
- ඊළඟ කුඩා කොටස Word එකට දෙනවා.
    
- ආයෙත් Chrome එකට දෙනවා.
    

මේ වැඩේ කෙතරම් වේගයෙන් වෙනවද කියනවා නම්, අපිට ඒක දැනෙන්නේ නැහැ. අපිට හිතෙන්නේ වැඩ දෙකම එක පාර සිද්ධ වෙනවා කියලා. මේකට තමයි **Concurrency** කියලා කියන්නේ.

### 2. Context Switching (වැඩ මාරු කිරීම)

CPU එක එක වැඩකින් තවත් වැඩකට මාරු වෙන ක්‍රියාවලියට තමයි **Context Switching** කියන්නේ. කලින් වැඩේ නතර කරපු තැන මතක තියාගෙන තමයි එයා අලුත් වැඩේට යන්නේ.

### 3. Parallelism (සැබෑ ලෙසම එකවර සිදුවීම)

අද කාලේ තියෙන පරිගණක වල **Multi-core CPUs** (උදා: Dual-core, Quad-core) තියෙනවා. මෙතනදී ඇත්තටම Core දෙකක් හෝ හතරක් තියෙන නිසා, සැබෑ ලෙසම වැඩ දෙකක් එකම තත්පරයේදී කරන්න පුළුවන්. ඒකට කියන්නේ **Parallelism** කියලා.
