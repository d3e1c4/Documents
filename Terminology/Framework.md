**Framework** එකක් යනු මෘදුකාංගයක් නිර්මාණය කිරීම සඳහා අවශ්‍ය වන මූලික සැකිල්ල හෝ ව්‍යුහයයි (**Foundation/Skeleton**). මෙය මෘදුකාංගයක් ගොඩනැගීමට අවශ්‍ය නීති රීති, මෙවලම් සහ පදනම සපයයි.

**Framework** එකක් යනු **සම්පූර්ණයෙන්ම නිම නොකළ නිවසක සැකිල්ලක් (Prefab House)** වැනිය. නිවසේ බිත්ති, වහලය සහ කාමර තිබිය යුතු තැන් කලින්ම තීරණය කර ඇත. ඔබට කළ හැක්කේ බිත්තිවල පාට තේරීම, ගෘහ භාණ්ඩ දැමීම වැනි දේ පමණි. ඔබට නිවසේ මූලික හැඩය (Structure) වෙනස් කිරීමට අපහසුය.

- **Inversion of Control (IoC):** මෙතනදී පාලනය තියෙන්නේ Framework එක අතේ. Framework එක විසින් හදලා දීලා තියෙන නීති සහ රාමුව ඇතුලේ තමයි ඔයා Code එක ලියන්න ඕනේ. ඔයාට ඕනේ වෙලාවට නෙවෙයි, Framework එකට අවශ්‍ය වෙලාවට තමයි ඒක ඔයාගේ Code එක Call කරන්නේ.

- **උදාහරණයක්:** මේක හරියට ඔයාට සම්පූර්ණයෙන්ම හදපු ආදර්ශ නිවසක් (Model House) දෙනවා වගේ වැඩක්. බිත්ති, කාමර ඔක්කොම වෙන් කරලා ඉවරයි. ඔයාට තියෙන්නේ ඒ දීලා තියෙන රාමුව ඇතුලේ ඉඳගෙන ඒවට පාට ගාන්න සහ බඩු තියන්න විතරයි. (උදාහරණ: Angular, Vue.js).

### Framework උදාහරණ

- **Java:** [[Spring Boot]] (Back-end සඳහා ඉතා ප්‍රසිද්ධයි). 
- **JavaScript:** [[React]] (Library එකක් වුවත් බොහෝ විට Framework එකක් ලෙස ක්‍රියා කරයි), [[Angular]], [[Vue]].
- **Python:** [[Django]], [[Flask]].
- **PHP:** [[Laravel]].

### Framework vs Library

මේ දෙකේ ප්‍රධානම වෙනස Software Engineering වල හඳුන්වන්නේ **"Inversion of Control"** (පාලනය ආපසු හැරීම) කියලා. සරලවම කිව්වොත් මේක තීරණය වෙන්නේ _"කවුද කාව Call කරන්නේ?"_ කියන එක මතයි.

| **ලක්ෂණය (Feature)** | **Library**                                                    | **Framework**                                         |
| -------------------- | -------------------------------------------------------------- | ----------------------------------------------------- |
| **ක්‍රියාකාරිත්වය**  | ඔයාගේ Code එකෙන් Library එක Call කරනවා.                        | Framework එකෙන් ඔයාගේ Code එක Call කරනවා.             |
| **පාලනය (Control)**  | පාලනය තියෙන්නේ Developer අතේ.                                  | පාලනය තියෙන්නේ Framework එක අතේ.                      |
| **නිදහස (Freedom)**  | ගොඩක් වැඩියි. ඔයාට ඕනේ විදිහට Architecture එක හදාගන්න පුළුවන්. | අඩුයි. Framework එකේ නීති සහ ව්‍යුහයට යටත් වෙන්න ඕනේ. |
| **ප්‍රමාණය සහ වේගය** | සාමාන්‍යයෙන් ප්‍රමාණයෙන් කුඩායි. වේගවත්.                       | ප්‍රමාණයෙන් විශාලයි. ගොඩක් දේවල් ඇතුලත් වෙනවා.        |
| **උදාහරණ**           | React, jQuery                                                  | Angular, Vue.js, Django                               |
සරලවම සාරාංශයක් විදිහට කිව්වොත්, Library එකක් කියන්නේ ඔයාගේ වැඩේ ලේසි කරගන්න පාවිච්චි කරන **Tool** එකක්. Framework එකක් කියන්නේ ඔයා වැඩ කරන **Environment** එක (පරිසරය) සහ නීති පද්ධතියක්.
